# 41_작업지시서 — 후보 앱 구조 정리

파일명: 41_작업지시서_후보앱_구조정리_v1.2.md
작성일: 2026-07-12
목적: CheckFlow 빈 골격 삭제 + 후보앱_목록.md 생성으로 GuideBook 구조 표준화

---

## 배경

현재 아래 두 폴더가 존재하나 실체가 없는 빈 골격이다.

```
C:\Obsidian\GuideBook\04_CheckFlow\
C:\Obsidian\GuideBook\GuideBook_생성기\apps\CheckFlow\
```

CheckFlow는 아이디어 단계(후보 앱)이며 개발이 확정되지 않았다.

확정된 GuideBook 운영 원칙:
```
폴더 있음  = 개발 중 (실제 프로젝트)
폴더 없음  = 후보 (문서로만 관리)
```

따라서 빈 골격을 삭제하고 후보앱_목록.md로 관리한다.

---

## STEP 0 — 삭제 전 내용 및 Git 상태 확인

### 0-1. 폴더 내용 확인
```powershell
"=== 04_CheckFlow\ ==="
Get-ChildItem -LiteralPath "C:\Obsidian\GuideBook\04_CheckFlow" -Recurse -Force |
    Select-Object FullName, Length | Format-Table -AutoSize

"=== apps\CheckFlow\ ==="
Get-ChildItem -LiteralPath "C:\Obsidian\GuideBook\GuideBook_생성기\apps\CheckFlow" -Recurse -Force |
    Select-Object FullName, Length | Format-Table -AutoSize
```

### 0-2. Git 추적 상태 확인
```powershell
cd C:\Obsidian\GuideBook
git -c core.quotepath=false ls-files -- "04_CheckFlow/" "GuideBook_생성기/apps/CheckFlow/"
git -c core.quotepath=false status --short
```

판단 기준:
- 실제 내용 파일(md, html, py 등) 없음 → STEP 1 진행
- 실제 내용 파일 있음 → 중지하고 보고한다
- Git 추적 파일이 있으면 삭제 후 Git에도 반영 필요함을 보고에 포함한다

완료 후 보고 형식:
```
1. 04_CheckFlow\ 내부 파일 목록
2. apps\CheckFlow\ 내부 파일 목록
3. 실제 내용 파일 존재 여부
4. Git 추적 파일 존재 여부
5. 삭제 진행 가능 여부
```

완료 후 중지하고 다음 지시를 기다린다.

---

## STEP 1 — CheckFlow 폴더 전체 삭제 (배포 + 생성기)

STEP 0에서 삭제 가능 확인 후 두 폴더를 한 번에 삭제한다.

```powershell
# 배포 폴더 삭제
Remove-Item -LiteralPath "C:\Obsidian\GuideBook\04_CheckFlow" -Recurse -Force
"04_CheckFlow 삭제 후 존재: $(Test-Path -LiteralPath 'C:\Obsidian\GuideBook\04_CheckFlow')"

# 생성기 앱 폴더 삭제
Remove-Item -LiteralPath "C:\Obsidian\GuideBook\GuideBook_생성기\apps\CheckFlow" -Recurse -Force
"apps\CheckFlow 삭제 후 존재: $(Test-Path -LiteralPath 'C:\Obsidian\GuideBook\GuideBook_생성기\apps\CheckFlow')"

# 삭제 후 전체 구조 확인
"=== GuideBook\ 루트 ==="
Get-ChildItem -LiteralPath "C:\Obsidian\GuideBook" | Select-Object Name | Format-Table -AutoSize

"=== apps\ ==="
Get-ChildItem -LiteralPath "C:\Obsidian\GuideBook\GuideBook_생성기\apps" | Select-Object Name | Format-Table -AutoSize
```

완료 후 보고 형식:
```
1. 04_CheckFlow 삭제 결과 (False여야 정상)
2. apps\CheckFlow 삭제 결과 (False여야 정상)
3. GuideBook\ 루트 현재 구조
4. apps\ 현재 구조
```

완료 후 중지하고 다음 지시를 기다린다.

---

## STEP 2 — 후보앱_목록.md 생성

STEP 1 완료 후 진행한다.

생성 위치:
```
C:\Obsidian\GuideBook\99_공통\후보앱_목록.md
```

내용:
```markdown
# GuideBook 후보 앱 목록

## 운영 원칙

- 폴더 있음 = 개발 중 (실제 프로젝트)
- 폴더 없음 = 후보 (이 문서로만 관리)

프로젝트 승인 시 구현계획서 → 작업지시서 → apps/{App}/ 생성 순서로 진행한다.

---

## 후보 앱

| 앱 | 상태 | 승인 | 생성일 | 비고 |
|---|---|---|---|---|
| CheckFlow | 후보 | 미승인 | - | 아이디어 단계, 미착수 |

---

## 앱 생명주기

아이디어
    ↓ (이 문서에 등록)
후보 앱
    ↓ (구현계획서 작성)
구현계획서
    ↓ (작업지시서 실행)
apps/{App}/ 생성 + Document Set 작성
    ↓ (Generator 실행 + Phase B)
Phase B 완료
    ↓ (GuideBook_배포/{App}/ 생성)
배포
```

생성 명령:
```powershell
$content = @"
# GuideBook 후보 앱 목록

## 운영 원칙

- 폴더 있음 = 개발 중 (실제 프로젝트)
- 폴더 없음 = 후보 (이 문서로만 관리)

프로젝트 승인 시 구현계획서 → 작업지시서 → apps/{App}/ 생성 순서로 진행한다.

---

## 후보 앱

| 앱 | 상태 | 승인 | 생성일 | 비고 |
|---|---|---|---|---|
| CheckFlow | 후보 | 미승인 | - | 아이디어 단계, 미착수 |

---

## 앱 생명주기

아이디어 → 후보 등록 → 구현계획서 → 작업지시서 → apps 생성 → Phase B → 배포
"@

Set-Content -LiteralPath "C:\Obsidian\GuideBook\99_공통\후보앱_목록.md" -Value $content -Encoding UTF8
"생성 완료: $(Test-Path -LiteralPath 'C:\Obsidian\GuideBook\99_공통\후보앱_목록.md')"
```

완료 후 보고 형식:
```
1. 생성된 파일 경로
2. 파일 내용 출력
3. 저장 완료 여부
```

완료 후 중지하고 다음 지시를 기다린다.

## 제약

- generator.py 수정 금지
- analyzer.py 수정 금지
- 실제 내용 파일(md, html, py 등)이 있으면 삭제 중단
- Remove-Item은 반드시 -LiteralPath 사용
- CheckFlow 폴더를 새로 생성하지 않는다
- 각 STEP 완료 후 반드시 중지하고 다음 지시를 기다린다
