## 현재 워크플로우 (v1.3 최신)

### 전체 사이클 한눈에 보기

```
[기획·설계]
  ① 채팅창 대화 → 요구사항 검토
  ② Claude: {순번}_구현계획서_v{버전}.md 생성
  ③ Claude: {순번}_작업지시서_v{버전}.md 생성

[파일 이동]
  ④ 다운로드 → C:\Obsidian\GuideBook\99_공통\ 이동

[구현 — 방식 선택]
  ⑤-A 수동: 사용자가 guide.md 직접 작성 → generator.py 실행
  ⑤-B 자동: Claude Code가 소스 폴더 분석
             → analyzer.py → guide.md 자동 생성 → generator.py 실행

  ⑥ 사용자: 작업지시서 프롬프트 블록 복사 → Claude Code 입력창 붙여넣기
  ⑦ 사용자: Claude Code 결과 복사 → 채팅창 붙여넣기 → 검증

[시행착오 기록]
  ⑧ Claude: 시행착오_기록.md 누적 업데이트 (덮어쓰기, 내용은 누적)
  ⑨~⑪ 수정 작업지시서 생성 → Claude Code 재실행 → 반복

[Git]
  ⑫ Regression PASS 확인
  ⑬ git add -A → git commit -m "{prefix}: {내용}" → git push
      커밋 단위: 앱 하나 완료 시
      prefix: feat: / fix: / docs: / chore:

[인수인계]
  ⑭ Claude: {순번}_HANDOVER_v{버전}.md 생성
  ⑮ 다운로드 → 99_공통\ 이동
  ⑯ 새 채팅창 열기 → MD 파일 첨부
  ⑰ "첨부한 인수인계 파일 읽고 이전 내용 파악해줘. 이어서 진행할게." 입력

[반복·완성]
  ⑱ ①~⑰ 반복 (프로젝트 완성까지)
  ⑲ Phase B 완료 → git tag v1.2 → 최종 가이드북 생성
```

### 파일명 규칙

```
{순번}_{내용}_v{버전}.{확장자}

순번: 시간 순서 (1, 2, 3...)
버전: 소규모 변경 v1.0→v1.1 / 대규모 변경 v1.0→v2.0

예외 (순번·버전 없이 덮어쓰기 + 내용 누적):
  - 시행착오_기록.md
  - 워크플로우_가이드.md
```

### 누적 관리 파일 규칙

| 파일 | 방식 | 이유 |
|------|------|------|
| 시행착오_기록.md | 덮어쓰기 + 내용 누적 | 번호 이어서 계속 추가 |
| 워크플로우_가이드.md | 덮어쓰기 + 이력 누적 | 현재 절차 + 변경 이력 함께 보관 |
| {순번}_작업지시서 | 새 파일 생성 | 버전별 보관 필요 |
| {순번}_HANDOVER | 새 파일 생성 | 채팅마다 순번 증가 |

### 빠른 참조 — 자주 쓰는 요청 문구

| 상황 | 채팅창 입력 |
|------|-------------|
| 계획서 만들기 | `지금까지 논의한 내용으로 구현계획서 만들어줘` |
| 지시서 만들기 | `작업지시서 만들어줘. Claude Code에 바로 붙여넣을 프롬프트 포함해줘` |
| 오류 기록 | `시행착오 기록 추가해줘. 오류: [내용]` |
| 인수인계 | `인수인계 파일 만들어줘. 새 채팅창에서 이어서 진행할 거야` |
| 새 채팅 시작 | `첨부한 인수인계 파일 읽고 이전 내용 파악해줘. 이어서 진행할게` |
| Commit/Push | `git add -A && git commit -m "docs: finalize {앱} content baseline" && git push` |
| 최종 완성 | `버전 A(개발자용)와 버전 B(사용자용) 가이드북 만들어줘` |
| 워크플로우 업데이트 | `워크플로우 가이드 업데이트해줘. 변경 내용: [내용]` |

### v1.0 — 2026-07-06 최초 작성

**배경**
학원 앱 가이드북 생성이 반복 업무 → 자동화 필요.
BAT + Python 방식 채택 (학원 Windows 환경, 설치 불필요).

**결정 사항**
- PWA vs BAT 검토 → BAT + Python 선택 (나중에 PWA 확장 가능 구조 유지)
- 폴더명: `GuideBook_Generator` → `GuideBook` 단축
- 메뉴: 하드코딩 → `apps/` 폴더 자동 스캔으로 동적 생성
- Python 표준 라이브러리만 사용 (외부 설치 불필요)

**확정된 구조**
```
GuideBook_생성기/
├── GuideBook_Generator.bat   ← 2단계 동적 메뉴
├── generator.py              ← guide.md → SOP + 튜토리얼 변환
├── apps/{앱}/input/output/   ← 앱별 분리
└── templates/                ← 공통 HTML 템플릿
```

**17단계 개발 사이클 확정**
채팅창(기획) → 작업지시서 → Claude Code(구현) → 검증 → 시행착오 기록
→ 반복 → 인수인계 → 새 채팅 → 반복 → 최종 완성

### v1.1 — 2026-07-06 소스 분석 자동화 방식 추가

**배경**
기존: guide.md를 사용자가 직접 작성 → 시간 소요, 누락 가능성.

**추가 사항**
- `analyzer.py` 신규 파일 추가
  - 소스 폴더 경로 + 분석 대상 폴더 목록 입력
  - 지원 확장자: .md .html .js .jsx .py .bat .sql .css .json
  - 파일 1개당 최대 500줄 읽기 (대용량 안전 처리)
  - 인코딩: UTF-8 우선, 실패 시 CP949 시도
- BAT 메뉴에 4번(소스 분석), 5번(분석+생성 한번에) 추가

**앱별 소스 경로 매핑 확정**
| 앱 | 소스 경로 | 분석 폴더 |
|----|-----------|-----------|
| KingTestMaker | C:\Obsidian\복테메이커\ | docs, electron, html, log |
| PrintFlow | C:\Obsidian\PrintFlow\ | Archive, command, log, MVP, sync, WORK, 배포 |
| Dooly | C:\Obsidian\Dooly\ | 00_System, 01_Project, 02_Claude_Project, 03_Claude_Code, 05_RAG, api, docs |

**⑤단계 분기 추가**
```
⑤-A 수동: 사용자가 guide.md 직접 작성
⑤-B 자동: analyzer.py → guide.md 자동 생성
```
