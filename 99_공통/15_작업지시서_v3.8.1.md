## 문제 정의

현재 Regression은 모드에 관계없이 모든 항목을 검사한다.

```
sop 모드로 실행
  → Tutorial HTML 없음 → Core FAIL
  → <pre><code> 없음  → Renderer FAIL (콘텐츠가 없어서)
  → Overall FAIL
```

이 결과는 **Generator 버그가 아님에도 FAIL로 표시**된다.
의도된 미생성(sop 모드) vs 실제 오류를 구분할 수 없다.

---

## 목표

mode에 따라 검사 항목을 3가지로 구분한다.

| 판정 | 의미 | 표시 |
|------|------|------|
| PASS | 검사 완료, 정상 | ✓ |
| FAIL | 검사 완료, 이상 | ✗ |
| N/A  | 해당 모드에서 생성 안 함 (정상) | - |

### 모드별 적용 규칙

```
both 모드     → SOP + Tutorial 모두 검사
sop 모드      → SOP 검사 / Tutorial 항목 → N/A
tutorial 모드 → Tutorial 검사 / SOP 항목 → N/A
```

### Overall 판정 변경

```
현재: Core + Renderer + App 중 하나라도 FAIL → Overall FAIL
변경: FAIL 항목이 하나라도 있으면 → FAIL
      FAIL 없고 N/A만 있으면    → PASS (with N/A)
      전부 PASS                 → PASS
```

## STEP 3. check() 헬퍼에 N/A 지원 추가

기존 check() 함수를 아래로 교체한다.

```python
def check(results, fails, label, condition, fail_msg=None, na=False):
    """
    na=True  → N/A 처리 (해당 모드에서 생성 안 함)
    na=False → 정상 PASS/FAIL 판정
    """
    if na:
        results.append(f'- {label}')   # N/A: 하이픈으로 표시
    elif condition:
        results.append(f'✓ {label}')
    else:
        results.append(f'✗ {label}')
        fails.append(fail_msg or label)
```

## STEP 4. Core Regression — mode-aware 적용

```python
# sop 모드면 Tutorial 항목을 N/A로 처리
tutorial_na = (mode == 'sop')
sop_na      = (mode == 'tutorial')

check(core_results, core_fails, 'guide.md 읽기',
      input_mtimes['guide.md'] != '파일 없음', 'guide.md 없음')
check(core_results, core_fails, 'rule.md 읽기',
      input_mtimes['rule.md']  != '파일 없음', 'rule.md 없음')
check(core_results, core_fails, 'faq.md 읽기',
      input_mtimes['faq.md']   != '파일 없음', 'faq.md 없음')
check(core_results, core_fails, 'SOP HTML 생성',
      sop_html and os.path.exists(sop_html), 'SOP HTML 생성 실패',
      na=sop_na)
check(core_results, core_fails, 'SOP MD 생성',
      sop_md   and os.path.exists(sop_md),   'SOP MD 생성 실패',
      na=sop_na)
check(core_results, core_fails, 'Tutorial HTML 생성',
      tutorial_html and os.path.exists(tutorial_html), 'Tutorial HTML 생성 실패',
      na=tutorial_na)
```

## STEP 5. Renderer Regression — mode-aware 적용

sop 모드가 아닐 때 SOP HTML이 없으면 N/A 처리한다.

```python
rend_results, rend_fails = [], []
if sop_na:
    rend_results.append('- Renderer 검사 (sop 미생성, N/A)')
elif sop_html and os.path.exists(sop_html):
    html = open(sop_html, encoding='utf-8').read()
    check(rend_results, rend_fails, '코드블록 렌더링 (<pre><code>)',
          '<pre><code>' in html, 'SOP에 <pre><code> 없음')
    check(rend_results, rend_fails, '수평선 렌더링 (<hr>)',
          '<hr '        in html, 'SOP에 <hr> 없음')
    check(rend_results, rend_fails, 'H4 렌더링 (<h4>)',
          '<h4'         in html, 'SOP에 <h4> 없음')
    check(rend_results, rend_fails, '인라인 코드 렌더링 (<code>)',
          '<code>'      in html, 'SOP에 인라인 <code> 없음')
else:
    rend_results.append('- SOP HTML 없어 Renderer 검사 건너뜀')

rend_pass = not rend_fails
```

## STEP 6. Overall 판정 변경

N/A만 있고 FAIL이 없으면 PASS로 처리한다.

```python
# FAIL 항목이 하나라도 있으면 FAIL, 없으면 PASS
all_fails = core_fails + rend_fails + app_fails
overall = 'FAIL' if all_fails else 'PASS'

# N/A 항목 존재 여부 체크 (표시용)
has_na = any(r.startswith('- ') for r in
             core_results + rend_results + app_results)
overall_display = f'PASS (N/A 포함)' if (overall == 'PASS' and has_na) else overall
```

터미널 출력과 Report에서 `overall` 대신 `overall_display` 사용.
