## STEP 2. generator.py 수정 — read_input_files() 함수 추가

guide.md를 읽는 기존 로직을 아래처럼 변경한다.

### 변경 전 (단일 파일 읽기)
```python
guide_path = get_base_dir() / "apps" / app_name / "input" / "guide.md"
guide_content = guide_path.read_text(encoding="utf-8")
```

### 변경 후 (다중 파일 자동 병합)

아래 함수를 generator.py 상단 함수 영역에 추가:

```python
def read_input_files(app_name):
    """
    input/ 폴더의 md 파일을 아래 순서로 읽어 하나로 합친다.
    읽는 순서: guide.md → rule.md → faq.md
    파일이 없으면 건너뜀 (오류 없음)
    """
    input_dir = get_base_dir() / "apps" / app_name / "input"
    files = ["guide.md", "rule.md", "faq.md"]
    combined = []
    found = []

    for fname in files:
        fpath = input_dir / fname
        if fpath.exists():
            try:
                content = fpath.read_text(encoding="utf-8")
            except UnicodeDecodeError:
                content = fpath.read_text(encoding="cp949")
            combined.append(content)
            found.append(fname)

    print(f"  읽은 파일: {', '.join(found)}")
    return "\n\n---\n\n".join(combined)
```

### 호출부 변경

기존에 guide.md를 직접 읽던 부분을 찾아서 아래로 교체:

```python
guide = read_input_files(app_name)
```

기존 guide_content 변수를 사용하던 곳은
read_input_files() 반환값을 그대로 사용하면 됨.

## 파일 저장 원칙 (8가지)

1. 빠짐없이 저장한다
2. 중복 없이 저장한다
3. 원본 파일은 유지하고 덮어쓰지 않는다
4. 시간 순서대로 저장한다
5. 파일을 열어보지 않아도 내용을 알 수 있도록 저장한다
6. 나중에 검색하기 쉽게 저장한다
7. 파일 출처를 알 수 있게 저장한다
8. 숙제와 테스트를 명확히 구분하여 저장한다

## 스캔 후 파일명 규칙

### A. 숙제
규칙: `원본파일명 + 학생이름 + -횟수`

숙제의 횟수는 같은 자료를 학생이 몇 번째 푼 것인지를 나타낸다.

| 예시 | 의미 |
|------|------|
| A 김철수-1.pdf | A 자료, 김철수, 첫 번째 숙제 |
| A 김철수-2.pdf | A 자료, 김철수, 두 번째 숙제 |
| B 김철수-1.pdf | B 자료, 김철수, 첫 번째 숙제 |

### B. 테스트
규칙: `원본파일명 + 학생이름 + 틀린문제수 + 개`

테스트의 숫자는 시험에서 처음 틀린 문제의 전체 개수를 나타낸다.

| 예시 | 의미 |
|------|------|
| C 김철수 2개.pdf | C 자료, 김철수, 최초 시험에서 2문제 틀림 |
| D 김철수 3개.pdf | D 자료, 김철수, 최초 시험에서 3문제 틀림 |

### 원본 파일 예시 (4개인 경우)
```
A.hwp  B.pdf  C.pdf  D.hwp
```
스캔 후 저장:
```
A 김철수-1.pdf  (숙제 첫 번째)
A 김철수-2.pdf  (숙제 두 번째)
B 김철수-1.pdf  (숙제 첫 번째)
C 김철수 2개.pdf  (테스트, 2문제 틀림)
D 김철수 3개.pdf  (테스트, 3문제 틀림)
```

| 증상 | 원인 | 해결 |
|------|------|------|

Q. 자료 DB 폴더를 바꿨는데 자료가 그대로예요.
A. 폴더 변경 후 반드시 **저장 & 재스캔** 버튼을 클릭해야 합니다.

## STEP 5. 각 앱 빈 파일 미리 생성

앞으로 동일한 구조를 재사용할 수 있도록 아래 빈 파일을 생성한다.
이미 파일이 있으면 건너뜀.

```
apps\KingTestMaker\input\rule.md  → 내용: # KingTestMaker 업무 규칙\n\n(추후 작성)
apps\KingTestMaker\input\faq.md   → 내용: # KingTestMaker FAQ\n\n(추후 작성)
apps\Dooly\input\rule.md          → 내용: # Dooly 업무 규칙\n\n(추후 작성)
apps\Dooly\input\faq.md           → 내용: # Dooly FAQ\n\n(추후 작성)
apps\CheckFlow\input\rule.md      → 내용: # CheckFlow 업무 규칙\n\n(추후 작성)
apps\CheckFlow\input\faq.md       → 내용: # CheckFlow FAQ\n\n(추후 작성)
```

Python으로 생성하는 방법:

```python
from pathlib import Path

base = Path(r"C:\Obsidian\GuideBook\GuideBook_생성기\apps")

targets = [
    ("KingTestMaker", "rule", "# KingTestMaker 업무 규칙\n\n(추후 작성)"),
    ("KingTestMaker", "faq",  "# KingTestMaker FAQ\n\n(추후 작성)"),
    ("Dooly",         "rule", "# Dooly 업무 규칙\n\n(추후 작성)"),
    ("Dooly",         "faq",  "# Dooly FAQ\n\n(추후 작성)"),
    ("CheckFlow",     "rule", "# CheckFlow 업무 규칙\n\n(추후 작성)"),
    ("CheckFlow",     "faq",  "# CheckFlow FAQ\n\n(추후 작성)"),
]

for app, kind, content in targets:
    path = base / app / "input" / f"{kind}.md"
    if not path.exists():
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(content, encoding="utf-8")
        print(f"  생성: {path}")
    else:
        print(f"  건너뜀(이미 존재): {path}")
```
