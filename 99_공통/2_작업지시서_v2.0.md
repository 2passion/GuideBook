## STEP 1. analyzer.py 생성

C:\Obsidian\GuideBook\GuideBook_생성기\analyzer.py 를 아래 스펙으로 만들어줘.

### analyzer.py 역할
- 소스 폴더 경로 + 분석할 하위 폴더 목록을 입력받음
- 지정 폴더의 파일들을 읽어 내용 추출
- 추출 내용으로 guide.md 자동 생성
- 출력 위치: GuideBook_생성기/apps/{앱이름}/input/guide.md

### analyzer.py 처리 순서

1. 폴더 스캔
   - 지정된 하위 폴더들의 파일 목록 수집
   - 읽기 가능한 텍스트 파일만 처리
   - 지원 확장자: .md, .txt, .html, .js, .jsx, .py, .bat, .sql, .css, .json

2. 파일 읽기 우선순위
   1순위: .md 파일 (docs/, README 포함)
   2순위: .html, .jsx 파일 (화면 구성 파악)
   3순위: .js, .py 파일 (기능 로직 파악)
   4순위: .bat, .sql 파일 (실행 절차 파악)
   - 파일 1개당 최대 500줄만 읽음 (대용량 파일 안전 처리)
   - 인코딩: UTF-8 우선, 실패 시 CP949 시도

3. 내용 추출 항목
   - 앱 목적: README.md 또는 docs/ 내 첫 번째 md 파일 상단
   - 주요 메뉴·버튼명: html/jsx에서 버튼 텍스트, 메뉴 이름 추출
   - 업무 절차: command/, docs/ 의 순서 있는 목록 추출
   - 오류·주의사항: log/ 의 오류 패턴 추출
   - FAQ: 자주 등장하는 질문 패턴

4. guide.md 생성
   형식:
   ---
   # {앱이름} 가이드
   생성일: {날짜}
   소스 경로: {소스 루트}
   분석 폴더: {폴더 목록}
   분석 파일 수: {N}개

   ## 앱 개요
   {README 또는 docs에서 추출}

   ## 주요 화면 및 메뉴
   {html/jsx에서 추출한 화면명, 버튼명}

   ## 업무 절차
   {command/docs에서 추출한 절차}

   ## 주의사항 및 오류
   {log에서 추출한 내용}

   ## 분석 파일 목록
   {읽은 파일 경로 목록}
   ---

### analyzer.py 코드 조건
- Python 표준 라이브러리만 사용
- 파일 읽기 실패 시 건너뛰고 계속 진행 (오류로 중단 금지)
- 진행 상황을 한국어로 출력 ("파일 읽는 중: xxx.md")
- 완료 후 생성된 guide.md 경로 출력
- 인코딩: UTF-8 (BOM 없음)

### analyzer.py 실행 방법 (BAT에서 호출)
python analyzer.py [앱이름] [소스루트경로] [폴더1,폴더2,폴더3]

예시:
python analyzer.py KingTestMaker "C:\Obsidian\복테메이커" "docs,electron,html,log"
python analyzer.py PrintFlow "C:\Obsidian\PrintFlow" "command,log,WORK,배포"
python analyzer.py Dooly "C:\Obsidian\Dooly" "00_System,01_Project,docs,api"


## STEP 2. GuideBook_Generator.bat 에 소스 분석 메뉴 추가

  5. 출력 폴더 열기
  0. 이전 메뉴

  4. 소스 폴더 분석 → guide.md 자동 생성  ← 신규
  5. 소스 분석 + A+B 생성 (한번에)         ← 신규
  ─────────────────────────
  6. 입력 폴더 열기
  7. 출력 폴더 열기
  0. 이전 메뉴
