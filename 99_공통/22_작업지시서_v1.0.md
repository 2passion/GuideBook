## 작업 목표

이번 작업은 검사 규칙 코드 네이밍을 간결하게 개선하는 리팩터링이다.

Mark-A/Mark-B 접두어를 HW/TEST로 교체한다.
Reference의 자료분류 코드(A/B/C/D)는 그대로 참조한다.
내용을 삭제하는 것이 아니라, 네이밍을 개선한다.

Generator 기능은 수정하지 않는다.

---

## 변경 대상

### rule_PrintFlow.md — 검사 규칙 네이밍 교체

#### Before (현재)
```
검사는 숙제 검사(Mark-A)와 테스트 검사(Mark-B) 두 가지로 구분한다.

Mark-A. 숙제 검사 대상
  Mark-A_A-01 교재 검사
  Mark-A_A-02 복테 검사
  Mark-A_A-03 오답노트 검사
  Mark-A_B-01 숙제 프린트 검사
  Mark-A_C-01 내신모의 검사
  Mark-A_C-02 기출 검사

Mark-B. 테스트 검사 대상
  Mark-B_B-02 테스트 프린트 검사
  Mark-B_C-01 내신모의 검사
  Mark-B_C-02 기출 검사
  Mark-B_D-01 진단평가 검사
  Mark-B_D-02 입학테스트 검사
```

#### After (개선)
```
검사는 숙제 검사(HW)와 테스트 검사(TEST) 두 가지로 구분한다.

HW. 숙제 검사 대상
  HW_A-01 교재 검사
  HW_A-02 복테 검사
  HW_A-03 오답노트 검사
  HW_B-01 숙제 프린트 검사
  HW_C-01 내신모의 검사
  HW_C-02 기출 검사

TEST. 테스트 검사 대상
  TEST_B-02 테스트 프린트 검사
  TEST_C-01 내신모의 검사
  TEST_C-02 기출 검사
  TEST_D-01 진단평가 검사
  TEST_D-02 입학테스트 검사
```

## 네이밍 변경 원칙

```
Mark-A  →  HW   (Homework, 숙제 검사)
Mark-B  →  TEST (Test, 테스트 검사)

이유:
  Reference가 이미 A/B/C/D 대분류를 정의한다.
  Rule은 역할(HW/TEST)만 추가하면 된다.
  Mark-A_C-01 → HW_C-01 로 간결해진다.
```

## 작업 순서

1. rule_PrintFlow.md 수정
   → 검사 구분 서두 교체 (Mark-A/B → HW/TEST)
   → 검사 코드 교체 (Mark-A_X-XX → HW_X-XX)
   → 검사 코드 교체 (Mark-B_X-XX → TEST_X-XX)

2. Regression 실행
   python generator.py PrintFlow both
