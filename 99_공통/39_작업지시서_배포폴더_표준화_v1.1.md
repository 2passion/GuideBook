# 39_작업지시서 — 배포 폴더 표준화

파일명: 39_작업지시서_배포폴더_표준화_v1.1.md
작성일: 2026-07-12
이전 버전: v1.0 (삭제 순서 및 검증 절차 보완)
목적: 배포 폴더(0X_{App}\)를 HTML 전용으로 표준화하고 최신본으로 교체

---

## 배포 폴더 표준 (확정)

```
배포 폴더 표준

- SOP HTML과 Tutorial HTML만 배포한다.
- SOP Markdown은 배포 폴더에서 제거한다.
- SOP Markdown 내부 생성물은 apps\{App}\output\ 에 유지한다.
- guide/rule/faq/reference SSOT는 apps\{App}\input\ 에 유지한다.
- 배포 HTML은 apps\{App}\output\ 의 최신본을 복사한다.
- 원본 HTML이 없으면 삭제·복사를 진행하지 않는다.

역할 구분:
  input\   = SSOT (guide/rule/faq/reference Markdown)
  output\  = 내부 생성 결과 (SOP.md / SOP.html / 튜토리얼.html)
  0X_App\  = 사용자 배포 결과 (SOP.html / 튜토리얼.html 만)
```

---

## 대상 경로

```
배포 폴더: C:\Obsidian\GuideBook\
생성기:    C:\Obsidian\GuideBook\GuideBook_생성기\apps\
```

---

## 각 STEP 공통 실행 순서

```
① 원본 HTML 2개 존재 확인
   → 없으면 중지하고 누락 파일 보고
② 대상 폴더 존재 확인
③ SOP.html 복사 및 덮어쓰기
④ 튜토리얼.html 복사 및 덮어쓰기
⑤ 원본·대상 파일 크기 비교 (복사 성공 확인)
⑥ 복사 성공 확인 후 배포 폴더의 SOP.md 삭제
⑦ 최종 폴더 목록 출력
⑧ 중지 후 사용자 확인 대기
```
