# 19_작업지시서_v3.10.md
주제: 입력 파일명 표준화
작성일: 2026-07-09
Generator 목표 버전: v3.10

---

## 배경

기존 입력 파일명 `guide.md / rule.md / faq.md`는
앱 이름이 포함되지 않아 나중에 참고·수정 시 헷갈릴 수 있다.

파일명 규칙을 아래와 같이 표준화한다.

```
{type}_{AppName}.md

type    : 소문자  (guide / rule / faq / reference)
AppName : PascalCase  (PrintFlow / KingTestMaker / Dooly / CheckFlow)
```

Generator 코드는 매핑 테이블 없이 app_name만으로 파일명을 생성한다.

---

## 작업 범위

### 1. 기존 input 파일 rename

아래 3개 앱의 input 폴더에서 파일명을 일괄 변경한다.

```
apps/PrintFlow/input/
  guide.md  → guide_PrintFlow.md
  rule.md   → rule_PrintFlow.md
  faq.md    → faq_PrintFlow.md

apps/KingTestMaker/input/
  guide.md  → guide_KingTestMaker.md
  rule.md   → rule_KingTestMaker.md
  faq.md    → faq_KingTestMaker.md

apps/Dooly/input/
  guide.md  → guide_Dooly.md
  rule.md   → rule_Dooly.md
  faq.md    → faq_Dooly.md
```

> ⚠️ 기존 파일은 삭제하지 않고 rename만 수행한다.
> ⚠️ rename 대상 이름과 동일한 파일이 이미 존재하면 자동 덮어쓰기 금지.
>    해당 파일명과 앱명을 보고한 뒤 작업을 중단한다.
> ⚠️ rename 성공 후 반드시 Regression을 실행하여 Generator가 새 파일명으로
>    실제 읽는지 확인한다. rename만 성공으로 완료 처리하지 않는다.
> images/ 폴더는 변경하지 않는다.

### 2. generator.py 수정

입력 파일 읽기 로직을 아래와 같이 변경한다.

**변경 전:**
```python
guide_path     = input_dir / "guide.md"
rule_path      = input_dir / "rule.md"
faq_path       = input_dir / "faq.md"
```

**변경 후:**
```python
guide_path     = input_dir / f"guide_{app_name}.md"
rule_path      = input_dir / f"rule_{app_name}.md"
faq_path       = input_dir / f"faq_{app_name}.md"
reference_path = input_dir / f"reference_{app_name}.md"  # optional
```

**reference 처리 규칙:**
- `reference_{app_name}.md` 파일이 존재하면 읽어서 SOP에 포함한다.
- 파일이 없으면 건너뛰고 정상 동작한다. (오류 없음)

**터미널 출력 변경:**

reference 없을 때:
```
읽은 파일: guide_PrintFlow.md, rule_PrintFlow.md, faq_PrintFlow.md
           reference_PrintFlow.md — 없음 (optional)
```

reference 있을 때:
```
읽은 파일: guide_PrintFlow.md, rule_PrintFlow.md, faq_PrintFlow.md,
           reference_PrintFlow.md
```

> reference가 없어도 터미널에 `없음 (optional)` 으로 표시한다.
> 사용자가 "왜 안 읽혔는지" 혼동하지 않도록 항상 존재 여부를 출력한다.

**Regression Core 항목 변경:**
```
✓ guide_{app_name}.md 읽기
✓ rule_{app_name}.md 읽기
✓ faq_{app_name}.md 읽기
- reference_{app_name}.md (optional — 없으면 N/A, PASS 유지)
✓ SOP HTML 생성
✓ SOP MD 생성
✓ Tutorial HTML 생성
```

### 3. analyzer.py 수정

출력 파일명을 아래와 같이 변경한다.

**변경 전:**
```python
output_path = get_base_dir() / "apps" / app_name / "input" / "guide.md"
```

**변경 후:**
```python
output_path = get_base_dir() / "apps" / app_name / "input" / f"guide_{app_name}.md"
```

터미널 출력도 함께 변경:
```
[완료] guide_KingTestMaker.md 생성: apps/KingTestMaker/input/guide_KingTestMaker.md
```

### 4. GuideBook_Generator.bat 수정

앱 관리 > 새 앱 생성 시 생성되는 기본 파일명을 변경한다.

**변경 전:**
```bat
> "apps\!new_app_name!\input\guide.md"
```

**변경 후:**
```bat
> "apps\!new_app_name!\input\guide_!new_app_name!.md"
```

### 5. Regression

rename 성공 후 3개 앱 모두 both 모드로 Regression 실행.

```
python generator.py PrintFlow both
python generator.py KingTestMaker both
python generator.py Dooly both
```

**PASS 기준:**
| 앱 | 기준 | 비고 |
|----|------|------|
| PrintFlow | 15/0/0 또는 15/0/1 | PASS |
| KingTestMaker | 14/0/0 또는 14/0/1 | PASS |
| Dooly | 14/0/0 또는 14/0/1 | PASS |

> ⚠️ N/A는 FAIL이 아니다.
> reference_{AppName}.md 가 없으면 해당 항목은 N/A로 처리되며 PASS를 유지한다.
> 15/0/1 은 "15 PASS / 0 FAIL / 1 N/A" 이므로 정상이다.

**터미널 출력으로 파일명 변경 검증 (필수):**

각 앱 실행 시 아래 형식으로 출력되어야 한다.
```
읽은 파일: guide_PrintFlow.md, rule_PrintFlow.md, faq_PrintFlow.md
           reference_PrintFlow.md — 없음 (optional)

읽은 파일: guide_KingTestMaker.md, rule_KingTestMaker.md, faq_KingTestMaker.md
           reference_KingTestMaker.md — 없음 (optional)

읽은 파일: guide_Dooly.md, rule_Dooly.md, faq_Dooly.md
           reference_Dooly.md — 없음 (optional)
```

## 제약 조건

- SSOT 원칙 준수: 입력 파일 내용은 수정하지 않는다. 파일명만 변경한다.
- rename 시 동일 이름 파일이 이미 존재하면 자동 덮어쓰기 금지. 보고 후 중단.
- rename 성공 후 반드시 Regression 실행하여 새 파일명 읽기 확인. rename만으로 완료 처리 금지.
- reference 파일 없음 시 오류 없이 동작. Core에서 N/A 처리, PASS 유지.
- N/A는 FAIL이 아니다. Regression Summary에서 N/A 수가 늘어나도 FAIL 아님.
- 기존 output 파일명(PrintFlow_SOP_v1.0.html 등)은 변경하지 않는다.
- 구현 개선 허용 원칙 적용 가능: 동일 목표 동작 유지 범위 내에서 더 나은 구현 허용.
