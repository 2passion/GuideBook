## PrintFlow 가이드북 스펙 (Claude Code 구현용)

### 확인된 실제 화면 (스크린샷 8장)

| 번호 | 화면명 | 주요 UI 요소 |
|------|--------|-------------|
| 1 | 기본 설정 | 사용자 이름 입력+저장, 자료DB폴더 탐색+**저장 & 재스캔**(파란 버튼), 동기화폴더 탐색, 데이터 백업(지금 백업/탐색기에서 열기), 역할 안내(원장/강사/조교) |
| 2 | 사용 앱 목록 | Dooly, King Test Maker, PrintFlow 3개 앱 확인 |
| 3 | 학생 관리 | 반/학교/학년/담당강사 필터 4개, 학생 목록 테이블(이름·반·학교·학년·담당강사·수정·삭제), + 학생 추가 버튼 |
| 4 | 자료 열람 > 폴더 탐색 | 좌측 4단계 트리(01테스트>1테스트(초등)>단원평가(초3~초6)>초3-1단평), 우측 파일 목록(PDF 뱃지+파일명+열기 버튼), 전체선택 체크박스 |
| 5 | 자료 열람 > 내 목록/계획 (상단) | 저장된 목록 리스트(날짜순/학년순 정렬), 선택된 목록의 파일 목록(HWP 뱃지+열기), 목록 삭제·이름변경·파일추가 버튼 |
| 6 | 자료 열람 > 내 목록/계획 (하단) | 계획 등록 섹션: 목록선택 드롭다운, 반 선택 입력, 학생 선택("3명 선택됨"), 수업 예정일 날짜 입력, "3명 × 8개 파일 = 24개 Task 생성 예정" 안내, **계획 등록 (24개)** 파란 버튼 |
| 7 | 계획 열람 (완료 상태) | 달력 월별 뷰, 날짜별 프린터 아이콘+숫자, 하단 목록: 목록명(파란색)+파일수+학생수+학생이름, **완료 ↩** 버튼, 완료된 항목은 초록 배경 |
| 8 | 계획 열람 (미완료 상태) | 미완료 항목은 주황/베이지 배경, **프린트** 파란 버튼, 파일 목록+열기 버튼 |

### PrintFlow SOP 목차 (표준 구조 적용)

```
# PrintFlow 업무 매뉴얼 (SOP)
대상: 신규 조교·강사 | 작성일: 2026-07 | 버전: v1.0

## 0. PrintFlow란?
  기존 방식(텍스트파일→폴더직접탐색) vs PrintFlow 방식 비교

## 1. 업무 흐름 한눈에 보기
  강사K → 자료선택 → 목록생성 → 학생선택 → 계획등록
  → (동기화) →
  조교P → 새로고침 → 계획열람 → 출력 → 완료
  → 조교Q/R → 미완료 이어서 처리

## A. 앱 설정
  A-1. 기본 설정 (사용자명, 자료DB폴더, 동기화폴더)
       ⚠️ 자료DB 변경 시 반드시 "저장 & 재스캔" 클릭
  A-2. 학생 등록 (반·학교·학년·담당강사)

## B. 강사(K) 사용법
  B-1. 자료 열람 > 폴더 탐색으로 자료 찾기
  B-2. 목록에 추가 → 내 목록/계획 탭에서 목록 이름 지정
  B-3. 계획 등록: 목록선택 → 반선택 → 학생선택 → 날짜선택 → 등록
       (N명 × M파일 = Task 수 자동 계산)

## C. 조교(P·Q·R) 사용법
  C-1. 출근 즉시: 새로고침 버튼 클릭 (최신 계획 수신)
  C-2. 계획 열람: 달력에서 오늘 날짜 클릭 → 목록 확인
  C-3. 파일 열기 → 학생 수에 맞게 출력 → 완료 버튼 클릭

## D. 조교 간 인수인계
  미완료 항목은 다음 조교가 새로고침 후 이어서 처리
  완료 버튼을 누른 항목은 초록색으로 표시됨

## E. 자주 하는 실수 (6가지)
  증상→원인→해결 표

## F. FAQ (4가지)
```

### PrintFlow HTML 튜토리얼 STEP 구성

```
[강사용 탭] / [조교용 탭] 선택

=== 조교용 (신규 조교 교육 우선) ===

STEP 1: 앱 실행 후 새로고침
  - 화면: 기본설정 화면 (screenshot 1)
  - 행동: 좌측 상단 "새로고침" 버튼 클릭
  - ✅ "아직 새로고침 전" → "새로고침 완료" 문구 확인

STEP 2: 계획 열람 클릭
  - 화면: 계획 열람 달력 (screenshot 7)
  - 행동: 좌측 메뉴 "계획 열람" 클릭

STEP 3: 오늘 날짜 클릭
  - 화면: 달력에서 날짜 선택
  - 행동: 오늘 날짜 클릭 → 하단에 목록 펼쳐짐 확인
  - ✅ 미완료(주황) 항목이 오늘 할 일

STEP 4: 파일 열기 및 출력
  - 화면: 계획 열람 미완료 상태 (screenshot 8)
  - 행동: "열기" 버튼 → 프린터 → 학생 수만큼 출력

STEP 5: 완료 버튼 클릭
  - 화면: 완료 버튼 강조
  - 행동: 출력 후 반드시 "완료" 클릭
  - ✅ 초록 배경으로 바뀌면 성공
  - ⚠️ 완료 안 누르면 다음 조교가 중복 출력

STEP 6: 완료
  - 핵심 3가지 요약 체크리스트
  - 새로고침 → 계획 열람 → 완료 버튼

=== 강사용 ===

STEP 1: 자료 열람 > 폴더 탐색
  - 화면: screenshot 4
  - 행동: 좌측 트리에서 단원 선택 → 우측 파일 체크 → "목록에 추가"

STEP 2: 내 목록/계획 탭 → 목록 이름 지정
  - 화면: screenshot 5

STEP 3: 계획 등록 (학생·날짜 선택)
  - 화면: screenshot 6
  - 행동: 반선택 → 학생선택 → 수업예정일 → "계획 등록 (N개)" 클릭
```

## 사용자 선호 / 특이사항

- **튜토리얼 형태 선호**: 단계별 이전/다음 버튼 방식을 강하게 선호 (문서 형태보다 효과적)
- **SOP 구조**: "설명서"가 아닌 "업무 매뉴얼" 형태, 역할별(강사/조교) 분리
- **실제 UI 기반**: 버튼명·메뉴명은 스크린샷에서 확인한 실제 명칭 그대로 사용
- **파일명 규칙 중시**: `{앱이름}_{문서종류}_v{버전}` 형식 엄수
- **다중 앱 관리**: 현재 4개 앱, 앞으로 추가 예정 → 번호 폴더로 관리
- **개발 환경**: VS Code + Claude Code (Extension) 사용
- **배포 방식**: PWA (BAT 방식 대신 선택)
