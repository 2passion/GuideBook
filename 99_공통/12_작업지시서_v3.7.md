## STEP 2. md_to_html() 함수 개선

generator.py의 md_to_html() 함수를 아래 순서로 개선한다.

### 2-1. 코드블록 처리 (``` ... ```)

현재: ``` 가 그대로 `<p>` ``` `</p>` 로 출력됨
변경: ``` ... ``` 블록 전체를 `<pre><code>...</code></pre>`로 변환

처리 방식 — 멀티라인 블록이므로 줄 단위 처리가 아닌 전처리(pre-pass)로 구현:

```python
import re

def preprocess_code_blocks(text):
    """
    ```...``` 블록을 <pre><code>...</code></pre>로 변환.
    언어 힌트(```python, ```bash 등)는 제거하고 내용만 유지.
    """
    def replacer(m):
        code = m.group(1)
        # HTML 특수문자 이스케이프
        code = code.replace('&', '&amp;')
        code = code.replace('<', '&lt;')
        code = code.replace('>', '&gt;')
        return f'<pre><code>{code}</code></pre>'

    # ``` 언어힌트(옵션) 와 닫는 ``` 사이 내용 캡처
    pattern = r'```[^\n]*\n(.*?)```'
    return re.sub(pattern, replacer, text, flags=re.DOTALL)
```

md_to_html() 함수 시작 부분에서 preprocess_code_blocks()를 먼저 호출:
```python
def md_to_html(text):
    text = preprocess_code_blocks(text)
    # ... 기존 줄 단위 처리 계속
```

### 2-2. 수평선 처리 (`---`)

현재: `<p>---</p>` 로 출력됨
변경: `<hr>` 로 변환

줄 단위 처리에서 아래 조건 추가:
```python
# --- 로만 이루어진 줄 → <hr>
if line.strip() == '---':
    html_lines.append('<hr style="border:none;border-top:1px solid #e5e7eb;margin:20px 0;">')
    continue
```

주의: H2 언더라인(===, ---) 과 구분 필요.
순수하게 `---` 만 있는 줄만 `<hr>` 처리.
`## ` 로 시작하는 줄은 이미 먼저 처리되므로 충돌 없음.

### 2-3. H4 처리 (`####`)

현재: H1~H3만 처리, H4 미지원
변경: `#### 제목` → `<h4>제목</h4>` 추가

기존 H3 처리 아래에 추가:
```python
elif line.startswith('#### '):
    html_lines.append(f'<h4>{line[5:].strip()}</h4>')
    continue
```

CSS에도 h4 스타일 추가:
```css
h4 { color: #4b5563; margin-top: 16px; font-size: 14px; }
```

### 2-4. 인라인 코드 처리 (`` `text` ``)

현재: `` `text` `` 가 그대로 노출됨
변경: `` `text` `` → `<code>text</code>`

줄 단위 처리 후 인라인 변환 함수 추가:
```python
import re

def apply_inline(text):
    """인라인 마크다운 변환 (볼드, 인라인 코드 등)"""
    # 인라인 코드: `text` → <code>text</code>
    text = re.sub(r'`([^`]+)`', r'<code>\1</code>', text)
    # 볼드: **text** → <strong>text</strong> (기존 처리 통합)
    text = re.sub(r'\*\*([^*]+)\*\*', r'<strong>\1</strong>', text)
    return text
```

## STEP 3. Sticky Tab 적용

generate_sop()의 HTML body 구조에서
top-bar와 tab-bar를 하나의 sticky 컨테이너로 묶는다.

### 3-1. HTML 구조 변경

변경 전:
```html
<div class="top-bar"> ... </div>

<div class="tab-bar"> ... </div>
```

변경 후:
```html
<div class="sticky-header">
  <div class="top-bar"> ... </div>
  <div class="tab-bar"> ... </div>
</div>
```

### 3-2. CSS 변경

기존 `.top-bar`의 sticky 관련 속성 제거하고
`.sticky-header`로 이동:

```css
/* 기존 top-bar에서 제거할 속성 */
/* position:sticky; top:0; background:white; z-index:10 → sticky-header로 이동 */

.top-bar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 20px 0 16px;
  border-bottom: 1px solid #e5e7eb;
  /* sticky 관련 속성 제거 */
}

/* 새로 추가 */
.sticky-header {
  position: sticky;
  top: 0;
  background: white;
  z-index: 10;
  border-bottom: 1px solid #e5e7eb;
  margin-bottom: 24px;
}

/* tab-bar에서 border-bottom 제거 (sticky-header가 담당) */
.tab-bar {
  display: flex;
  gap: 0;
  border-bottom: 2px solid #e5e7eb;
  margin: 0;  /* sticky-header 안에서 margin 조정 */
}
```
