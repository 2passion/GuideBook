# PrintFlow 업무 규칙

## 파일 저장 원칙 (8가지)

1. 빠짐없이 저장한다
2. 중복 없이 저장한다
3. 원본 파일은 유지하고 덮어쓰지 않는다
4. 시간 순서대로 저장한다
5. 파일을 열어보지 않아도 내용을 알 수 있도록 저장한다
6. 나중에 검색하기 쉽게 저장한다
7. 파일 출처를 알 수 있게 저장한다
8. 숙제와 테스트를 명확히 구분하여 저장한다

---

## 스캔 후 파일명 규칙

### A. 숙제
규칙: `원본파일명 + 학생이름 + -횟수`

숙제의 횟수는 같은 자료를 학생이 몇 번째 푼 것인지를 나타낸다.

| 예시 | 의미 |
|------|------|
| A 김철수-1.pdf | A 자료, 김철수, 첫 번째 숙제 |
| A 김철수-2.pdf | A 자료, 김철수, 두 번째 숙제 |
| B 김철수-1.pdf | B 자료, 김철수, 첫 번째 숙제 |

### B. 테스트
규칙: `원본파일명 + 학생이름 + 틀린문제수 + 개`

테스트의 숫자는 시험에서 처음 틀린 문제의 전체 개수를 나타낸다.

| 예시 | 의미 |
|------|------|
| C 김철수 2개.pdf | C 자료, 김철수, 최초 시험에서 2문제 틀림 |
| D 김철수 3개.pdf | D 자료, 김철수, 최초 시험에서 3문제 틀림 |

### 원본 파일 예시 (4개인 경우)
```
A.hwp  B.pdf  C.pdf  D.hwp
```
스캔 후 저장:
```
A 김철수-1.pdf    (숙제 첫 번째)
A 김철수-2.pdf    (숙제 두 번째)
B 김철수-1.pdf    (숙제 첫 번째)
C 김철수 2개.pdf  (테스트, 2문제 틀림)
D 김철수 3개.pdf  (테스트, 3문제 틀림)
```

## 검사 규칙

검사는 숙제 검사(HW)와 테스트 검사(TEST) 두 가지로 구분한다.

### HW. 숙제 검사 대상
```
HW_A-01 교재 검사
HW_A-02 복테 검사
HW_A-03 오답노트 검사
HW_B-01 숙제 프린트 검사
HW_C-01 내신모의 검사
HW_C-02 기출 검사
```

### TEST. 테스트 검사 대상
```
TEST_B-02 테스트 프린트 검사
TEST_C-01 내신모의 검사
TEST_C-02 기출 검사
TEST_D-01 진단평가 검사
TEST_D-02 입학테스트 검사
```
